## 1. Zerodha Kite Connect Setup

### Why HTTPS is Required

Zerodha Kite Connect **requires HTTPS** for both the Redirect URL and Postback URL. HTTP is not accepted. You have two options depending on your environment:

---

### Option A — Local Development with ngrok (Recommended)

[ngrok](https://ngrok.com) creates a secure HTTPS tunnel to your local machine. This is the easiest way.

**Step 1 — Install ngrok**
```bash
# Download from https://ngrok.com/download, then:
sudo snap install ngrok          # Ubuntu/Debian
# or: brew install ngrok         # macOS
```

**Step 2 — Start your backend**
```bash
make backend   # starts FastAPI on http://localhost:8000
```

**Step 3 — Start an ngrok tunnel to port 8000**
```bash
ngrok http 8000
```

ngrok will print output like:
```
Forwarding   https://abc123.ngrok-free.app -> http://localhost:8000
```

**Step 4 — Use the ngrok HTTPS URL in Kite Connect:**

| Field            | Value                                                        |
|------------------|--------------------------------------------------------------|
| **Redirect URL** | `https://abc123.ngrok-free.app/api/auth/redirect`            |
| **Postback URL** | `https://abc123.ngrok-free.app/api/postback/`                |

> **Note**: Free ngrok URLs change every restart. To get a fixed URL, use a paid ngrok account or use Option B below.

---

### Option B — Local HTTPS with mkcert (Permanent/No Account Needed)

mkcert installs a local CA and creates trusted SSL certificates for localhost.

**Step 1 — Install mkcert**
```bash
sudo apt install libnss3-tools         # Ubuntu/Debian
sudo apt install mkcert
mkcert -install                        # Install local CA
```

**Step 2 — Generate certificate for localhost**
```bash
cd /mnt/Local\ Disk\ C/web-projects/zerodha-trade
mkcert localhost 127.0.0.1             # Creates localhost+1.pem and localhost+1-key.pem
```

**Step 3 — Run backend with SSL**

Install the `uvicorn` SSL dependency and update the Makefile backend target:
```bash
# In backend/.venv:
pip install uvicorn[standard]
```

Run with SSL:
```bash
cd backend && uvicorn app.main:app \
  --host 0.0.0.0 \
  --port 8000 \
  --ssl-keyfile ../localhost+1-key.pem \
  --ssl-certfile ../localhost+1.pem
```

**Step 4 — Use HTTPS localhost in Kite Connect:**

| Field            | Value                                              |
|------------------|----------------------------------------------------|
| **Redirect URL** | `https://localhost:8000/api/auth/redirect`         |
| **Postback URL** | `https://localhost:8000/api/postback/`              |

---

### Option C — Production Server (Deployed App)

If you deployed to a server (e.g., VPS with nginx + Let's Encrypt):

| Field            | Value                                                   |
|------------------|---------------------------------------------------------|
| **Redirect URL** | `https://your-domain.com/api/auth/redirect`             |
| **Postback URL** | `https://your-domain.com/api/postback/`                 |

Set up SSL with Let's Encrypt:
```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

Then proxy `/api` to port 8000 and `/` to port 3000 via nginx.

---

### Creating a Kite Connect App

1. Go to [Kite Connect Developer Console](https://developers.kite.trade/)
2. Create a new app. Use whichever URL matches your setup above.
3. Copy your **API Key** and **API Secret**
4. Add them to your `.env` file:
   ```
   TRADE_ZERODHA_API_KEY=your_api_key_here
   TRADE_ZERODHA_API_SECRET=your_api_secret_here
   ```
5. Update `NEXT_PUBLIC_API_URL` in `frontend/.env.local` if using ngrok:
   ```
   NEXT_PUBLIC_API_URL=https://abc123.ngrok-free.app
   ```

### Authentication Flow

1. Navigate to **Providers** page → Activate "zerodha" provider
2. Click the login URL generated by the backend
3. You log in on Zerodha's OAuth page
4. Zerodha redirects your browser to `GET /api/auth/redirect?request_token=xxx&status=success`
5. The backend exchanges `request_token` for an `access_token` (using your `api_secret`)
6. The backend redirects you to the frontend dashboard with a success message
7. You're now authenticated for live trading

> **Redirect URL** is for the OAuth login redirect (GET, browser redirect).  
> **Postback URL** is a separate webhook — Zerodha POSTs order status updates (COMPLETE, REJECTED, etc.) to it.

## 2. Paper Trading (Mock Mode)

Paper trading lets you practice **without risking real money**. The Mock provider simulates order matching, fills, positions, and P&L.

### How to Paper Trade

1. **Start the app** in mock mode (default):
   ```
   TRADE_DEFAULT_PROVIDER=mock  # in .env
   make backend   # Start backend on :8000
   make frontend  # Start frontend on :3000
   ```

2. **Go to Providers** → Make sure "mock" provider is active

3. **Go to Paper Trading** page → Click "Start Paper Trading"
   - Set your virtual capital (default: ₹10,00,000)
   - Sample data for 20 popular NSE stocks is loaded automatically
   - Stocks include: RELIANCE, TCS, INFY, HDFCBANK, ICICIBANK, SBIN, etc.

4. **Place manual orders** → Go to Orders page
   - Select exchange (NSE)
   - Search for a symbol (type to autocomplete from loaded instruments)
   - Choose BUY/SELL, quantity, order type
   - Orders fill instantly at the sample price ± slippage

5. **Monitor positions** → Dashboard shows open positions and P&L

### Mock Trading vs Real Data

The mock system uses **sample prices** for 20 major NSE stocks. These are approximate market prices to give you realistic practice. When you connect Zerodha (live mode), the app uses real-time market data from Zerodha's API.

For more realistic paper trading with real historical data, switch to Zerodha provider and use Kite Connect's historical data API. The mock engine is designed for learning the workflow.

## 3. Creating & Running Strategies

### Available Strategy Types

| Strategy | Description |
|----------|-------------|
| **SMA Crossover** | Buys when fast SMA crosses above slow SMA (bullish), sells when it crosses below |
| **RSI Mean Reversion** | Buys when RSI drops below oversold level, sells when RSI rises above overbought |

### How to Create a Strategy

1. Go to **Strategies** page
2. Click **"+ New Strategy"**
3. Select a strategy type (e.g., `sma_crossover`)
4. Configure parameters:
   - **Trading Symbol**: The stock to trade (e.g., RELIANCE)
   - **Instrument Token**: Numeric token (shown in mock instruments list)
   - **Fast/Slow Period**: Moving average lookback periods
   - **Quantity**: Shares per trade signal
5. Click **"Create Strategy"**
6. Click **"Start"** on the created strategy card

### Strategy Parameters (SMA Crossover)

| Parameter | Default | Description |
|-----------|---------|-------------|
| Fast SMA Period | 10 | Candles for fast moving average |
| Slow SMA Period | 20 | Candles for slow moving average |
| Instrument Token | — | Numeric token of the stock |
| Trading Symbol | — | e.g., RELIANCE |
| Exchange | NSE | Exchange name |
| Quantity | 1 | Shares per signal |

### Strategy Parameters (RSI Mean Reversion)

| Parameter | Default | Description |
|-----------|---------|-------------|
| RSI Period | 14 | Lookback for RSI calculation |
| Oversold | 30 | Buy when RSI drops below this |
| Overbought | 70 | Sell when RSI rises above this |
| Instrument Token | — | Numeric token of the stock |
| Trading Symbol | — | e.g., TCS |
| Exchange | NSE | Exchange name |
| Quantity | 1 | Shares per signal |

## 4. How Automated Trading Works

### The automation pipeline:

```
Market Data → Strategy → Signal → Order Manager → Risk Check → Broker Provider → Order
```

1. **Market data** flows in via WebSocket ticks or candle polling
2. **Strategies** process prices and compute indicators (SMA, RSI, etc.)
3. When conditions are met, strategies **emit signals** (BUY/SELL)
4. The **Order Manager** receives signals and creates order requests
5. The **Risk Manager** validates against limits (max loss, position size, etc.)
6. If approved, orders are placed via the active **Broker Provider**
7. In mock mode → orders fill in the mock engine
8. In live mode → orders go to Zerodha's real exchange

### Why is there a manual order form?

The manual order form on the Orders page is for:
- **Quick manual trades** when you spot an opportunity
- **Testing** that your provider connection works
- **Emergency** position management
- **Learning** before enabling automation

**Strategies automate this process.** Once a strategy is created and started, it generates orders automatically based on market data. No manual intervention needed.

## 5. Live Trading with Zerodha

### Prerequisites

1. Zerodha Demat account
2. Kite Connect subscription (₹2000/month for API access)
3. API Key and Secret configured in `.env`

### Steps

1. **Configure `.env`**:
   ```
   TRADE_DEFAULT_PROVIDER=zerodha
   TRADE_ZERODHA_API_KEY=your_key
   TRADE_ZERODHA_API_SECRET=your_secret
   ```

2. **Start the application**:
   ```
   make backend
   make frontend
   ```

3. **Activate Zerodha provider** on the Providers page

4. **Authenticate** via the Kite Connect OAuth flow

5. **Configure risk limits** on Settings page:
   - Max daily loss
   - Max order value
   - Max positions
   - Orders per minute rate limit

6. **Create strategies** on the Strategies page

7. **Start strategies** and monitor on Dashboard

### Risk Controls

**Always configure risk limits before live trading:**
- Set max daily loss to limit how much you can lose per day
- Set max order value to cap individual trade size
- Use the Kill Switch for emergency stop of all trading

## 6. How to Make Profit

> **Disclaimer**: Trading involves risk. Past performance doesn't guarantee future results.

### Getting Started (Recommended Path)

1. **Learn** → Start with paper trading to understand the platform
2. **Backtest** → Test strategies on sample data first
3. **Paper trade** → Run strategies with virtual capital for weeks
4. **Small live** → Start with small capital when confident
5. **Scale** → Gradually increase based on results

### Strategy Tips

- **SMA Crossover** works best in trending markets (stocks moving consistently up/down)
- **RSI Mean Reversion** works best in range-bound markets (stocks oscillating in a range)
- **Don't over-optimize** — if parameters only work on one stock, they'll likely fail on others
- **Diversify** — run strategies on multiple uncorrelated stocks
- **Control risk** — never risk more than 1-2% of capital per trade

### What this app helps with

- **Discipline** — strategies execute automatically, no emotional trading
- **Speed** — automated execution is faster than manual
- **Risk management** — built-in limits prevent catastrophic losses
- **Backtesting** — test ideas before risking real money
- **Monitoring** — real-time dashboard shows portfolio health

## 8. FAQ

**Q: What URL do I configure in Kite Connect?**
A: Zerodha requires **HTTPS**. Two different URLs:
- **Redirect URL** = `https://localhost:8000/api/auth/redirect` — where the browser goes after OAuth login (GET)
- **Postback URL** = `https://localhost:8000/api/postback/` — where Zerodha POSTs order status webhooks
- For local dev, use `make backend` (HTTPS by default) or ngrok
- For production, replace `localhost:8000` with your domain

**Q: How does paper trading work?**
A: The mock provider simulates order matching with virtual capital. Sample data for 20 NSE stocks is loaded when you create a session. All trades are simulated — no real money involved.

**Q: Can I use real Zerodha market data in mock mode?**
A: Not directly. Mock mode uses sample approximate prices. For real-time data, switch to the Zerodha provider (requires Kite Connect subscription). In future, we can add historical data replay from Zerodha for more realistic backtesting.

**Q: Why are orders manual? I thought strategies automate trading.**
A: They do! The manual order form is for ad-hoc trades. Strategies automatically generate and place orders when running. Create a strategy, start it, and it trades on its own.

**Q: How do I create a strategy?**
A: Strategies page → "+ New Strategy" → Pick type (SMA Crossover or RSI) → Configure parameters → Create → Start.

**Q: Is there a list of available symbols?**
A: Yes! In mock mode, 20 popular NSE stocks are pre-loaded. In the Orders page, the symbol field has autocomplete that searches available instruments. In live mode, all Zerodha-listed instruments are available.

**Q: How do I deactivate a provider?**
A: Providers page → Click "Deactivate" button on the active provider.

**Q: What is the Kill Switch?**
A: Emergency stop. Activating it immediately halts all strategy execution and blocks new orders. Use it if something goes wrong. Configure it in Settings.
